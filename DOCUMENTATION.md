## 1. Обзор проекта

### Назначение
Сайт-визитка стоматолога-ортопеда Никитиной Марины Георгиевны в Нижнем Новгороде. Сайт предоставляет информацию об услугах протезирования зубов, ценах и способах связи.

### Функциональность
- Информация об услугах (коронки, протезы, виниры)
- Прайс-лист с ценами
- Детальные страницы услуг с показаниями/противопоказаниями
- Страницы технологий с подробным описанием
- Адаптивный дизайн для мобильных устройств
- SEO-оптимизация для Яндекса и Google

### Стек технологий
- **Frontend:** Next.js 15 (React 19)
- **Стили:** Pure CSS (без Tailwind для избежания проблем с Build)
- **Хостинг:** RelaxDev (российский хостинг)
- **Git:** GitHub

## 2. Архитектура

### Структура маршрутов
```
/                           # Главная страница
├── /services               # Список всех услуг
│   ├── /[slug]             # Детальная страница услуги
│   │   ├── /condition/[slug]   # Страница показания/противопоказания
│   │   └── /variant/[slug]     # Страница варианта услуги
/technologies/[slug]       # Страницы технологий
```

### Рендеринг
- **Static Site Generation (SSG)** — все страницы генерируются при сборке
- Это обеспечивает максимальную скорость загрузки и SEO- дружественность

### Типы страниц
1. **Статические страницы** — главная, услуги, технологии
2. **Динамические страницы** — [slug] генерируются из данных в compile-time
3. **Страница 404** — обработка несуществующих URL

### 4.2 Страница услуги (app/services/[slug]/page.tsx)

**Назначение:** Детальная страница конкретной услуги.

**Структура:**
1. Описание услуги
2. Показания (ссылки на /services/condition/)

4. Варианты с ценами (ссылки на /services/variant/)
5. Уход
6. CTA с телефоном

**Генерирование мета-тегов:**
```typescript
export async function generateMetadata({ params }) {
  const service = SERVICES_DATA[slug];
  return {
    title: `${service.name} | Услуги стоматолога-ортопеда`,
    description: service.description.slice(0, 160),
    alternates: { canonical: `https://ortopednn.ru/services/${slug}` }
  };
}
```

1. Добавить текст в `app/services/conditions.ts`:
```typescript

  "novoe-pokazanie": {
    name: "Новое показание",
    description: "Описание...",
    symptoms: ["Симптом 1", "Симптом 2"],
    treatment: "Лечение..."
  }
```

2. Добавить маппинг в функцию `getConditionSlug()`:
```typescript
export function getConditionSlug(name: string): string {
  const map = {
    "Новое показание": "novoe-pokazanie"
  };
  return map[name] || ...;
}
```

3. Добавить страницу в `sitemap.xml`:
```xml
<url>
  <loc>https://ortopednn.ru/services/condition/novoe-pokazanie</loc>
  <priority>0.6</priority>
</url>
```

## 8. Развертывание

### 8.1 GitHub → RelaxDev

Проект настроен на автоматический деплой при push в ветку `develop`.

```bash
# После редактирования
git add -A
git commit -m "Описание изменений"
git push origin develop
```

RelaxDev автоматически:
1. Скачает код из GitHub
2. Выполнит `npm install`
3. Выполнит `npm run build`
4. Опубликует на ortopednn.ru

### 8.2 Проверка после деплоя

1. Открыть https://ortopednn.ru
2. Проверить главную страницу
3. Проверить страницы услуг

## 9. Частые вопросы FAQ

### Q: Как добавить новую услугу?

A: Добавить запись в `app/services/data.ts` по образцу существующих. Потом добавить slug в карту sitemap и commit+push.

### Q: Почему не используется Tailwind?

A: Tailwind вызывал ошибки сборки на RelaxDev. Pure CSS работает стабильно.

### Q: Как изменить телефон на сайте?

A: Изменить в `app/page.tsx`: `const CLINIC_PHONE = "+79202537317";`

### Q: Как добавить новую страницу показания?

A: 1) Добавить в `conditions.ts`, 2) Добавить маппинг, 3) Добавить в sitemap.

### Q: Сайт не обновляется после push?

A: Проверить статус Deploy на RelaxDev. Возможно, превышен лимит бесплатного плана (20 деплоев/месяц).

### Q: Как проверить sitemap?

A: Открыть https://ortopednn.ru/sitemap.xml в браузере.

### Q: Favicon не отображается в поиске?

A: Яндексу и Google требуется время (до 2-4 недель) для обнаружения и индексации favicon.

## 10. Лучшие практики

### 10.1 Работа с кодом

**Не удалять существующие маппинги** — это сломает ссылки на страницы.

**Проверять уникальность title/description** — дублирование негативно влияет на SEO.

**Тестировать локально** перед push:
```bash
npm run build  # Проверка сборки
npm run lint    # Проверка ошибок
```

### 10.2 SEO-рекомендации

1. **Региональная привязка** — настроена через geo-метатеги и Organization schema
2. **Обновление sitemap** — добавлять новые страницы
3. **Канонические URL** — всегда использовать canonical
4. **Адаптивность** — проверять на мобильных устройствах

### 10.3 Безопасность

- Не хранить секреты в коде
- Использовать `gitignore` для исключения `.env`
- Проверять внешние ссылки

### 10.4 Производительность

- Использовать SSG (Static Site Generation) — уже настроено
- Оптимизировать изображения (использовать SVG)
- Минизировать CSS — встроено в Next.js
